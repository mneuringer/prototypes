Activity Selector Patterns

Prototype demonstrating contextual activity information in dropdowns and displays

Pattern 1: Move Participant to Activity

Consistent context info + open spots. Search enabled for finding specific activities.

Current (single line)
Farm Camp (Week 1, Mon, Tue, Wed, Thu, Fri 9:00 AM - 3:00 PM) (30 open spots)
Proposed (multi-line)

Pattern 2: Apply to Specific Activities (Multi-select)

For registration questions or discounts. Same context info, no open spots.

Pattern 3: Activity Display (Read-only)

For student modals, payment pages, emails. Hover to see full details.

Key Improvements

Consistent context shown everywhere:
Season · Session or Dates · Days & Times · Grades · Instructor
Two display modes:
Rich contexts
Dropdowns, modals → hover/tooltip for details
Constrained contexts
Emails, mobile → details block shown nearby
Before:
  • All info crammed into one long line
  • Hard to scan and compare activities
  • Activity name gets lost in the details
  • Inconsistent info across different contexts
After:
  • Activity name always prominent
  • Same context info everywhere
  • Adapts to rich vs. constrained contexts
  • Search to quickly find activities
